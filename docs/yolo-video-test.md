# YOLO Video Test Clips

The local YOLO video test folder is:

```text
<DATA_ROOT>\yolovideotest
```

`<DATA_ROOT>` is the external ML data directory, defaulting to `../traffic-incident-data` unless `TRAFFIC_DATASETS_ROOT` is set.

For public GitHub release, these dataset-derived MP4 files are not committed to the repository. Keep or regenerate them locally under `<DATA_ROOT>\yolovideotest` when you want to run the YOLO demo.

It contains two short input clips:

- `mio_vehicle_short.mp4`: generated from labeled MIO-TCD localization validation images for vehicle detection / tracking checks.
- `rdd_damage_short.mp4`: generated from labeled RDD2022 validation images for road anomaly / debris checks.

It also contains annotated outputs generated by `yolo.infer_video`:

- `mio_vehicle_short.boxes.mp4`
- `rdd_damage_short.boxes.mp4`

These clips are intentionally short, around 8 seconds each. They are generated from existing image datasets rather than original continuous video. That makes them convenient for repeatable checks, but it also exposes useful model limitations: tracking quality is weak on non-continuous image sequences, and road-damage boxes may be low-confidence, missed, or imperfectly aligned.

## Dry-Run Commands

Vehicle / stopped-vehicle mode:

```powershell
cd <repo-root>
.\.venv\Scripts\python.exe -m yolo.infer_video `
  --mode vehicle `
  --weights <DATA_ROOT>\runs\mio-localization\mio-stage2-20260421-234643\weights\best.pt `
  --source <DATA_ROOT>\yolovideotest\mio_vehicle_short.mp4 `
  --base-url http://127.0.0.1:8000 `
  --camera-id CAM-YOLO-VIDEO-MIO `
  --highway-id E1 `
  --confidence 0.25 `
  --stop-seconds 1 `
  --cooldown-seconds 5 `
  --annotated-output <DATA_ROOT>\yolovideotest\mio_vehicle_short.boxes.mp4 `
  --dry-run
```

Road anomaly / debris mode:

```powershell
cd <repo-root>
.\.venv\Scripts\python.exe -m yolo.infer_video `
  --mode damage `
  --weights <DATA_ROOT>\runs\rdd2022\rdd-stage2-20260421-234643\weights\best.pt `
  --source <DATA_ROOT>\yolovideotest\rdd_damage_short.mp4 `
  --base-url http://127.0.0.1:8000 `
  --camera-id CAM-YOLO-VIDEO-RDD `
  --highway-id E1 `
  --confidence 0.25 `
  --frame-stride 1 `
  --cooldown-seconds 5 `
  --annotated-output <DATA_ROOT>\yolovideotest\rdd_damage_short.boxes.mp4 `
  --dry-run
```

Remove `--dry-run` when you want detections to be posted into the API and shown on the dashboard.

## Verified API Write Demo

Start the API and keep the seed service stopped so the YOLO records are easy to find:

```powershell
cd <repo-root>
docker compose up -d --build
docker compose stop seed
```

Run the RDD clip without `--dry-run`:

```powershell
.\.venv\Scripts\python.exe -m yolo.infer_video `
  --mode damage `
  --weights <DATA_ROOT>\runs\rdd2022\rdd-stage2-20260421-234643\weights\best.pt `
  --source <DATA_ROOT>\yolovideotest\rdd_damage_short.mp4 `
  --base-url http://127.0.0.1:8000 `
  --camera-id CAM-YOLO-VIDEO-RDD `
  --highway-id E1 `
  --confidence 0.25 `
  --frame-stride 1 `
  --cooldown-seconds 5 `
  --annotated-output <DATA_ROOT>\yolovideotest\rdd_damage_short.boxes.mp4
```

Then query the events:

```powershell
Invoke-RestMethod 'http://127.0.0.1:8000/events?camera_id=CAM-YOLO-VIDEO-RDD&sort_by=detected_at&order=desc&limit=10&offset=0' |
  ConvertTo-Json -Depth 12
```

- `CAM-YOLO-VIDEO-RDD:DEBRIS:20260426T141118178311:a44c447c`
- `CAM-YOLO-VIDEO-RDD:DEBRIS:20260426T141119377223:8545637c`
